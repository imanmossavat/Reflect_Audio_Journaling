# Weekly update — Reflect

**This week**
- **Modular pipeline** — restructured the search-and-answer pipeline into separate, swappable stages so each can be tested and improved on its own.
- **Evaluation + key result** — built the second setup with the dataset with multiplr notes to retrieve. Found the system was refusing to answer even when it had retrieved the right information. Fixing only the answer step raised accuracy from ~47% to ~67%, with retrieval unchanged, so the gain is cleanly isolated to generation.
- **Fixed a concurrency bug** — answering used to freeze the rest of the app. Now generation runs off the main thread (search stays responsive) and simultaneous requests queue in order, with a clear "waiting" state.
- **Onboarding** — added a guided tour (adapts to empty accounts, replayable) and a welcome entry on first launch.
